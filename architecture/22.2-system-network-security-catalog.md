# System, Network, and Security Catalog

## I/O, paths, files, and memory

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Io` | standard; planned; phase 2 | `Reader`, `Writer`, `Stream`, `Seek`, buffering, pipes, copy, limits; `Io.read`, `write`, `copy`, `buffer` | Bytes/Task; protocols are typed and backend-neutral; caller buffers reusable; copy reports bytes moved; blocking/suspension explicit; no filesystem authority or global streams |
| `Path` | standard; planned; phase 2 | lexical portable path value, components, join/parent/name/extension, normalize, relative resolution | Text; pure Pop; value operations do not touch storage; platform encoding preserved through typed native path representation; no existence or permission checks |
| `File` | standard/platform; planned; phase 2 | opaque `File.Handle`, `Open` options, metadata, permissions, links, mapping, locking; `open`, `read`, `write`, `atomicWrite`, `close` | Path/Io/Time + PLRI; convenience read materializes with a limit, streaming avoids it; root capability and symlink policy explicit; temporary files under `File.Temp`; no manager object |
| `Directory` | standard/platform; planned; phase 2 | create/remove/list/walk/watch, metadata, standard locations, temporary directories | Path/File/Time + PLRI; iteration streams entries; traversal limits and symlink policy required; watch semantics target-qualified; no implicit current-directory authority |
| `Memory` | standard/unsafe; planned; phase 1/6 | safe `Memory.View`, arenas/pools, mapped buffers; `Memory.copy`, `fill`; `Memory.Unsafe` raw addresses/pointers | Intrinsics/GC/PLRI; safe views bounds-checked and non-owning; pools expose retention; unsafe calls require explicit capability; no untracked managed pointer escape |

Virtual filesystems and storage roots implement typed `File.Access` and
`Directory.Access` function records passed explicitly. They do not impersonate
the process filesystem through global registration.

| `Environment` | standard/platform; planned; phase 2 | immutable environment snapshot and explicit get/set-for-child operations | Text + PLRI; reads/copies documented; names/encoding target-aware; secrets redact; mutation of ambient process environment is excluded from safe portable API |

## Tasks and concurrency

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Task` | standard/platform; planned; phase 3 | `Task<T>`, `CancelToken`, `Group`, deadline, spawn/join/race/select/yield/sleep; structured failure propagation | Time + PLRI scheduler; task creation may allocate; scope owns children; cancellation points explicit; no detached task by default or ambient context bag |
| `Channel` | standard; planned; phase 3 | bounded/unbounded typed channels, sender/receiver, select integration; `send`, `receive`, `trySend`, `close` | Task/Atomic; portable semantics; bounded is default for streams; allocation/copy/ownership documented; no untyped messages |
| `Actor` | standard/platform; planned; phase 4 | isolated local actors, exact typed mailboxes, actor references/replies, monitors, structured supervision, restart and shutdown policy | Task/Channel + scheduler and GC ownership contracts; messages are compiler-proven safe and copied into private actor ownership; mailboxes/restarts/cleanup are bounded; no shared mutable state, selective receive, marker-interface safety, or runtime type lookup |
| `Atomic` | standard/platform; planned; phase 3 | exact atomic integer/boolean/pointer-safe handles, memory order, fences, wait/notify | Intrinsics/PLRI; availability by target capability; operations allocate nothing; unsafe order combinations diagnosed; no managed raw pointers |
| `Actor` | standard/platform; planned; phase 4 | typed isolated `Ref<TMessage>`, `Inbox<TMessage>`, `Reply<T>`, supervisors, monitors, bounded mailboxes; `start`, `send`, `trySend`, `receive`, `reply` | Task/Channel/Codec + scheduler/GC isolation support; messages are statically proven copy-safe and copied into actor ownership; mailbox allocation, copying, suspension, stale incarnation, and failure are explicit; no symbolic registry, shared mutable actor state, or class hierarchy |
| `Cluster` | official/platform; planned; phase 8 | authenticated remote `Actor<TMessage>`, nodes, publish/spawn, delivery outcomes, remote supervision and test transports | Actor/Codec/Net/Crypto/Identity/Task; separately installed `Pop.Cluster`; bounded schema encoding and explicit partial failure; no location transparency, code shipping, string actor lookup, implicit retry, or exactly-once claim |

Locks, read/write locks, semaphores, events, barriers, task-local scoped values,
worker pools, parallel sequence operations, and deterministic schedulers live as
`Task.Lock`, `Task.Semaphore`, `Task.Event`, `Task.Barrier`, `Task.Scope`,
`Task.Pool`, `Task.Parallel`, and `Task.Test`. Native thread access is
`Platform.<Target>.Thread`; process concurrency remains `Process`.

`Process.Ipc` covers named/local pipes, typed local messages, process handles,
and shared-memory coordination. Byte transport uses `Io`; shared mappings use
`Memory.Shared`; Unix-domain sockets remain `Net.Unix`. `Process.Sandbox`
describes isolation profiles/capabilities and never claims portability beyond
the target manifest. `Process.Host` covers real operating-system daemon/service
registration and control, which justifies the host-specific noun without
creating a general application service abstraction.

## Network foundations and HTTP

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Net` | standard/platform; planned; phase 4 | `Address`, `Ip`, interfaces; `Net.Dns`, `Tcp`, `Udp`, `Unix`, `Tls`, `Quic`; connect/listen/resolve and transport function records | Socket/Task/Time/Crypto + PLRI/native; address values pure; I/O streams buffered with backpressure; DNS/TLS limits and cancellation required; no global transport registry |
| `Socket` | standard/platform; planned; phase 4 | opaque socket handle, options, local/remote address, accept/send/receive/close, multicast | Io/Net/Task + PLRI; system calls explicit; caller buffers reusable; option availability typed; raw sockets require unsafe capability |
| `Http` | official; planned; phase 4 | `Request`, `Response`, `Header`, `Method`, `Status`, cookies, forms, multipart, cache/proxy/auth/redirect/retry records; `send`, `serve`; `Http.Route`, `Client`, `Server`, `Test` | Net/Uri/Mime/Codec/Task/Telemetry/Crypto; streaming bodies and pools explicit; safe redirect/TLS/header/body limits; HTTP/1.1, 2, 3 adapters typed; `Client` only an opaque connection-pool resource, never a service object |
| `WebSocket` | official; planned; phase 4 | handshake, typed frame/message stream, ping/pong/close, compression options | Http/Task/Bytes; bounded frames and backpressure; origin/auth checks; no string-dispatched message routing |
| `Cluster` | official; planned; phase 4 | optional `Pop.Cluster` Package for authenticated distributed actor endpoints, publication, remote spawn, monitoring, placement, and typed delivery outcomes | Actor/Codec/Net/Crypto/Identity/Task; exact public message schemas and explicit node/transport capabilities; bounded encoding, transport, and mailbox admission; partial failure remains typed; no location transparency, code shipping, runtime symbol lookup, automatic retry, or exactly-once claim |

Server-sent events are `Http.Event`; forms and multipart are `Http.Form` and
`Http.Multipart`. Retry/backoff policy is a typed `Http.Retry` value. Test and
in-memory transports implement the same explicit transport function record.

## Cryptography, identity, and permissions

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Crypto` | standard/platform; planned; phase 4 | secure random, secret bytes, constant-time compare; `Crypto.Hash`, `Mac`, `Kdf`, `Aead`, `Sign`, `Exchange`, `Password`, `Cert`, `Key`, `Token`, `Unsafe` | Bytes/Time/Platform + vetted native/pure implementations; streaming and caller-buffer forms; secrets noncopying/redacted where possible; algorithms curated/versioned; no invented primitives or insecure defaults |
| `Identity` | official/platform; planned; phase 5 | principals, claims as typed schemas, credentials, authentication results; `Identity.Oauth`, `Oidc`, `Jwt`, `Passkey`, `Permission`, `KeyStore` | Crypto/Http/Locale/Platform; token/key material redacted; clock/audience/issuer validation mandatory; platform stores explicit; no string claim bags or ambient current user |

Capabilities are nominal typed values issued by hosts or explicit constructors.
Permission checks accept those values directly. FIPS/compliance modes are
target/package policies with declared approved algorithms and conformance
evidence, not a runtime boolean that silently changes semantics.
