# Core and Portable Library Catalog

## Reading the catalog

Every row is a placement contract, not an API completeness claim. `Pure` means
the primary implementation can be Pop source; `PLRI` means some operation needs
the runtime interface. Cost statements are intended contracts until benchmarks
stabilize a numeric budget.

## Values, numerics, and sequences

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Math` | standard; `prototype` arithmetic; phase 1 | scalar integer/float/decimal operations; `Math.Int`, `Math.Float`, `Math.Decimal`, `Math.Big`, `Math.Rational`, `Math.Complex`; `abs`, `min`, `max`, `clamp`, `gcd`, checked arithmetic | Intrinsics + pure Pop; fixed-width semantics portable; scalar calls allocate nothing; big values allocate explicitly; no random/statistics/linear algebra; numeric protocols are closed compiler/library contracts |
| `Random` | standard; planned; phase 1 | deterministic pseudo-random `State`, distributions needed by general programs; `seed`, `next`, `fill`, `shuffle` | Pure Pop plus entropy capability only for nondeterministic seeding; state is explicit; no global generator; cryptographic randomness belongs to `Crypto`; algorithms/version streams documented |
| `Guid` | standard; planned; phase 1 | 128-bit identifier value; `new`, `parse`, `format`, byte conversion | Pure value operations; generation takes random/clock policy explicitly; no allocation for binary value; UUID versions are named cases, not runtime strings |
| `Version` | standard; planned; phase 1 | semantic and tool version values; `parse`, `compare`, `matches` | Pure Pop; parsing bounded; prerelease/build text owns storage; package resolution policy stays in tooling |
| `Uri` | standard; planned; phase 1 | URI/URL values and views; `parse`, `resolve`, `encode`, `decode`; `Uri.Query` | Pure Pop; normalized/canonical forms allocate only when requested; parsing limits and scheme trust are explicit; HTTP transport belongs to `Http` |
| `Mime` | standard; planned; phase 1 | media type value, parameters, matching and extension lookup | Pure Pop; parsing bounded; registry data generated at toolchain build time; content sniffing is opt-in and belongs to format/media packages |
| `Sequence` | standard; `prototype`; phase 1 | `Iterable<T>`, `Iterator<T>`, ranges, lazy map/filter/fold/zip/window/chunk, sort/search/group/aggregate | Pure Pop + compiler iteration protocol; adapters lazy and allocation-free unless materialized; stable ordering documented; parallelism belongs to `Task`; no `Iter` alias |

Primitive values, tuples, result packs, records, `Option`, `Result`, equality,
ordering, hashing, copying, and conversion are language/prelude contracts rather
than namespace families. Collections use direct types (`Array`, `List`, `Map`,
`Set`, `Queue`, immutable variants) and `Sequence` algorithms. This avoids a
ceremonial `Collection` root.

## Text, bytes, and matching

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Text` | standard; `prototype` slicing and primitive formatting; phase 1 | UTF-8 `String`, borrowed `Text.View`, rune/grapheme iteration, split/join/trim/replace; `Text.Encoding`, `Escape`, `Template`, `Diff`, `Token`; parse/format helpers | Intrinsics + Unicode; views do not allocate, returned strings own storage; source `..`, interpolation, and `String(value)` use the closed ADR 0041 contract; indexes never split UTF-8; transcoding and diff output expose buffers/materialization; templates compile to typed parameter records; collation belongs to `Locale`; no mutable builder default, but reusable text buffers use `Bytes` with checked UTF-8 finish |
| `Unicode` | standard; planned; phase 1 | `Rune`, category/script/property tables, normalization, casing, grapheme/word/line segmentation, width | Generated Unicode data + pure Pop; version pinned in metadata; normalization allocates unless caller-buffer form used; locale-sensitive behavior belongs to `Locale` |
| `Bytes` | standard; planned; phase 1 | owned bytes, slices/views, reusable buffer, endian reads/writes, bit operations, hex/base32/base64 | Intrinsics + pure Pop; views allocate nothing; growth explicit; decoders expose output sizing and limits; cryptographic operations belong to `Crypto` |
| `Regex` | standard; planned; phase 2 | compiled `Regex.Pattern`, `compile`, `isMatch`, `find`, `captures`, `replace`, streaming search subset | Pure/backend-neutral engine or verified runtime primitive; compile allocates, matching declares worst-case policy and limits; no backtracking DoS default; extensions are compile-time syntax editions, not runtime handlers |
| `Glob` | standard; planned; phase 2 | path/text glob `Pattern`, `compile`, `matches`, deterministic expansion with explicit directory capability | Pure matcher; expansion uses `Directory`; bounded traversal and symlink policy required; no regex inheritance or shell execution |

## Formats and codecs

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Codec` | standard/official; planned; phase 2 | typed `Encoder<T>`, `Decoder<T>`, schema descriptors, generated adapters, binary framing; `Codec.Binary`, `Codec.Schema` | Text/Bytes/Io/Metadata; adapters statically resolved; caller-buffer and stream forms; schema recursion/size limits; vendor formats live in separate official Packages |
| `Json` | standard; planned; phase 2 | `encode`, `decode`, `read`, `write`, typed schema adapters; `Json.Value` only for deliberate schema-less data | Codec/Text/Io; pure Pop possible; direct decode avoids tree allocation; bounded depth/string/number sizes; duplicate-key policy explicit; extension through typed adapters |
| `Yaml` | standard; planned; phase 2 | safe YAML subset, typed encode/decode, event stream, authoring support for resources/settings | Codec/Text/Io; aliases, depth, nodes, and expansion bounded; unsafe tags disabled; direct typed decode preferred; no arbitrary object construction |
| `Xml` | standard; planned; phase 2 | typed encode/decode, pull events, bounded tree, namespaces, canonicalization, schemas in official extensions | Codec/Text/Io; DTD/entities/external resolution off; depth/text limits; streaming avoids tree allocation; no XPath/XSLT in standard core |
| `Csv` | standard; planned; phase 2 | records/typed rows, dialect record, streaming read/write, schema mapping | Codec/Text/Io; row buffers reusable; field/row limits; formula injection handling documented; spreadsheet behavior excluded |
| `Toml` | standard; planned; phase 2 | typed configuration decode/encode and source-preserving document edits | Codec/Text/Time; pure Pop; bounded parsing; duplicate keys rejected; application settings policy belongs to `Settings` |
| `Archive` | official; planned; phase 3 | `Archive.Entry`, streaming read/write; `Archive.Zip`, `Archive.Tar` | Io/Compress/Path; native optional; no implicit extraction; entry/size/count/path limits prevent traversal and bombs; codec additions are typed Package dependencies |
| `Compress` | official; planned; phase 3 | streaming compressors/decompressors; `Compress.Deflate`, `Gzip`, `Zstandard`, `Brotli`, `Lz4` | Io/Bytes plus vetted native or pure implementations; reusable buffers/backpressure; expansion limits mandatory; algorithm packages declare licensing/native requirements |

INI remains an official compatibility Package `Pop.Ini` only when a concrete
ecosystem need accepts its weak duplicate/key/type semantics. It is not a
reserved root in the standard catalog.

## Locale, resources, and time

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Locale` | standard/platform; planned; phase 2 | `Locale.Tag`, negotiation, collation, number/currency/unit/date formatting, calendars, time zones, plural/select rules; `Locale.Message` | Unicode/Time/Resource; CLDR/tz data generated and versioned; formatters reusable; no ambient current locale; platform locale discovery is explicit; custom data packs are typed/versioned |
| `Resource` | standard/tooling; planned; phase 2 | typed packaged text/binary assets, locale bundles, fallback, generated keys, `load`, `text`, `bytes` | Package metadata/Codec/Locale/Io; compile-time key validation; cache policy explicit; untrusted override limits; never `Table<String, String>` or runtime name lookup |
| `Time` | standard/platform; planned; phase 2 | `Instant`, `Duration`, `Date`, `DateTime`, `Zone`, `Clock`, `Deadline`; `now`, `measure`, `sleep`, `after`, interval/timer values | Pure values + PLRI clocks/timers; monotonic and wall clocks distinct; test clocks explicit; timer allocation/scheduling documented; locale formatting belongs to `Locale`; cron/job scheduling is official |

Localization YAML uses `resources/<tag>/<domain>.yaml`. The resource compiler
validates one default-locale schema, parameter types, plural/select coverage,
fallbacks, interpolation escaping, duplicate/missing keys, and rich-fragment
types. It generates nominal keys and parameter records. Runtime loaders accept
those identities, not strings; cache and override capabilities are explicit.
