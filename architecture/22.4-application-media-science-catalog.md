# Application, Media, and Science Catalog

## Messaging and application protocols

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Rpc` | official; bootstrap Package, APIs planned; phase 6 | typed request/response and streaming contracts; `Rpc.Schema`, `Client`, `Server`, `Test`; generated local/remote stubs | Codec/Http/Task/Telemetry; serialization and transport explicit; streaming backpressure; auth/deadline/size limits; `Client` is an opaque channel resource, not a service locator; owned by separately installed `Pop.Rpc` |
| `Message` | official; planned; phase 6 | broker-neutral typed envelope, topic identity, acknowledgement, delivery result; `Message.Queue`, `Stream`, `Broker`, `Test` | Codec/Task/Telemetry/Identity + explicit transport Package; batches and backpressure explicit; delivery semantics declared; no string payload/property bag or global broker registry |
| `Email` | official; planned; phase 6 | address/message/header/attachment values; MIME composition; `Email.Smtp`, `Imap`, `Test`; `send`, `read` | Mime/Net/Crypto/Codec/Task; attachments stream; header/size limits and injection prevention; credentials explicit; no ambient mail service |

Event sourcing, distributed logs, AMQP, MQTT, Kafka-compatible protocols, and
cloud queues are explicit Packages implementing `Message` contracts. They are
not hidden behind one lowest-common-denominator runtime provider.

Fonts and text shaping are `Graphics.Font` and `Graphics.Text`. Supported
families include font discovery, fallback, glyph runs, shaping, rasterization,
OpenType features, and variable fonts. Platform font access is explicit; font
parsers are bounded. Licensing-sensitive codecs remain separately distributed.

## User interfaces

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Ui` | official/platform; planned; phase 7 | immutable `View` values, state/update functions, effects, commands; `Ui.App`, `Window`, `Layout`, `Input`, `Access`, `Style`, `Theme`, `Text`, `Image`, `Animation`, `Clipboard`, `Drag`, `Dialog`, `Test` | Graphics/Image/Locale/Task/Telemetry/Platform; diff/layout/render cost measured; stable keys typed; platform effects explicit; no inheritance widget tree, global application singleton, or runtime string binding |

Desktop, mobile, web, and headless renderers are `Pop.Ui.Desktop`,
`Pop.Ui.Mobile`, `Pop.Ui.Web`, and `Pop.Ui.Headless` Packages implementing typed
backend records. Composable view functions remain ordinary Pop functions.
Accessibility is part of every view contract and cannot be added only by a
platform adapter. Navigation, focus, input, localization, themes, clipboard,
drag/drop, dialogs, and testing have backend-neutral state/events with explicit
platform availability.

Official scientific families planned beneath `Pop.Science` include
`Geospatial`, `Physics`, `Chemistry`, `Biology`, `Finance`, and `Frame`. They use
typed schemas/units and do not enter `Pop.Standard`. Sparse linear algebra,
accelerator kernels, and large datasets remain optional Packages.

## Artificial intelligence

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Ai` | official/platform; bootstrap Package, APIs planned; phase 8 | model/token/input/output values; `Ai.Model`, `Infer`, `Train`, `Token`, `Embed`, `Search`, `Generate`, `Tool`, `Evaluate`, `Data`, `Cache`, `Device`, `Test` | Pop.Data + Tensor/Codec/Http/Task/Telemetry/Crypto; local/remote runtimes explicit; streaming/backpressure and device transfer visible; credentials/redaction/limits required; no vendor-shaped root or untyped tool calls; owned by separately installed `Pop.Ai` |

Model formats and runtimes are typed Packages such as `Pop.Ai.Onnx` and
target/vendor bindings. Remote APIs implement generated typed transport records.
Tool calls use declared input/output schemas and resolved function identities,
not runtime function names. Reproducibility records model/content hashes,
runtime version, device, seed, settings, and dataset identity. Safety hooks are
explicit typed filters/evaluators and do not pretend to guarantee model safety.

## Native interoperability

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Ffi` | unsafe; prototype; architecture accepted by ADR 0081 and ADR 0082; implementation in progress; phase 6 | exact C/system ABI types, statically bound native libraries/symbols, read-only byte pins, owned ABI buffers, handles, callbacks, generated bindings; `Ffi.C`, `Buffer`, `Callback`, `Unsafe` | compiler/PLRI/Platform/Memory; canonical hashed link plans and layout fingerprints; every transition, borrow, and copy documented; callbacks rooted and thread rules explicit; no raw linker flags, shell commands, runtime symbol lookup, dynamic fallback, Pop object addresses, or untracked managed pointers |

Native libraries, operating-system APIs, GPU APIs, web-host bindings, and device
SDKs live in explicit platform or third-party Packages. Generated bindings carry
ABI hashes, target constraints, safety annotations, and reviewable source
metadata. `Memory.Unsafe` and `Ffi.Unsafe` never enter the prelude.

The first stable FFI is the ADR 0081 C-compatible surface. C++ libraries are
consumed through an exact C ABI or generated C-compatible shim; Pop Lang does
not expose C++ object layout, exception, overload, or mangling rules as a
portable source contract.
