# Data, Observability, and Tooling Catalog

## Databases and storage

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Data` | official; bootstrap Package, APIs planned; phase 5 | typed rows, columns, schemas, datasets, tabular frames, query values, nullability, bulk transforms; `Data.Row`, `Column`, `Schema`, `Set`, `Frame`, `Query` | Codec/Sequence/Task; pure typed values first; column/view buffers avoid copies; schemas resolved/generated; no universal dynamic row or property bag; owned by separately installed `Pop.Data` |
| `Sql` | official; planned; phase 5 | typed `Row`, `Column`, `Schema`, `Query`, parameters, connection/pool/cursor/transaction/savepoint; `open`, `prepare`, `query`, `execute`, `transaction`; `Sql.Migrate`, `Test` | Codec/Task/Time/Telemetry + explicit driver Package; rows stream and buffers may be reused; parameters never interpolated; cancellation/timeouts required; driver is typed function record, not provider hierarchy |
| `Store` | official/platform; planned; phase 5 | key-value, document, object/blob, embedded storage contracts; `Store.Key`, `Value`, `Batch`, `Cursor`; `Store.Kv`, `Document`, `Blob`, `Embedded`, `Cloud` | Codec/Io/Task/Telemetry + explicit engine Package; streaming values and batch costs documented; key/schema/transaction limits; no universal dynamic document type except explicit `Json.Value` |

Tabular data uses typed records and schemas. Data-frame/tensor analysis belongs
to `Science.Frame` and `Tensor`, not the persistence layer. Database nullability
maps to explicit optional fields. Bulk operations, isolation, retries, and
telemetry are typed option records. Transactions are resource scopes owned by
`Sql` or the relevant `Store` engine, never a top-level namespace.

## Telemetry

| Root | Tier/status/phase | Scope and public concepts | Dependencies, portability, cost, security, extension, exclusions |
| --- | --- | --- | --- |
| `Telemetry` | standard/official; planned; phase 5 | typed `Event`, `Field`, `Level`, correlation/scope; `Telemetry.Log`, `Trace`, `Metric`, `Crash`, `Test`; `log`, `span`, `count`, `measure` | Time/Task/Source; standard contracts + no-op/test sinks, official exporters; disabled paths have measurable near-zero budgets; buffering/sampling explicit; redaction required; no mandatory global singleton or string field bags |

`Telemetry.Log` defines structured events and filters; `Trace` defines spans,
links, propagation, and sampling; `Metric` defines counters, gauges, histograms,
units, and aggregation; `Crash` defines stack/source reports. Sinks/exporters are
explicit typed values or compile-time registered generated adapters. A scoped
application default may be passed at entry points, but APIs also accept an
explicit `Telemetry.Sink` and tests can capture it deterministically.

Dependency injection is not a library container. Applications compose immutable
configuration, capability values, function records, and real resource handles
directly. Compile-time metadata may generate wiring functions, but resolution is
typed and visible in generated code.

Source and syntax-tree APIs are owned by `Pop.Syntax`; diagnostics may remain a
separate tooling Package. Semantic queries, resolved AST, HIR, MIR, and compiler
incremental databases are excluded until separately designed. Public facades
cannot expose private compiler handles or permit source injection through
compile-time execution.

## Typed metadata lifecycle

Attributes are language syntax whose values are nominal library-defined types.
The compiler knows only reserved roles such as `@CompileTime`, usage/validator
contracts, and explicitly accepted code-generation/retention roles. Domain
Packages define attributes such as `@Command`, `@Description`, `@Category`,
`@Parameter`, `@Deprecated`, `@Stable`, `@RequiresCapability`, `@Serializable`,
`@Validate`, `@Route`, `@Message`, `@Generated`, and `@Test`.

Metadata values live in compiler-owned typed schemas during compilation.
Processors run under the deterministic compile-time budget/capability contract
and may emit structured diagnostics, adapter schemas, or generated bindings
through approved tooling APIs. Invalid target, duplicate, version, or
cross-attribute combinations are diagnosed by resolved identity. Runtime
retention is opt-in, versioned, visibility-preserving, and limited to generated
serializable projections. There is no symbol enumeration, reflection scan, or
get/call-by-name API.

ADR 0096 closes the first retained projection. A target generates one
same-visibility `TargetSchema: Codec.Schema<Target>` Item from canonical typed
`retained-adapters.popc`. The descriptor owns declaration ordinals, data labels,
closed types, full fingerprints, limits, and source provenance. ADR 0055 JSON
artifact control files may inventory its digest but cannot duplicate its
structure. Only reachable schema Items retain runtime adapter bodies/data.

`Package.Plugin` describes typed build/tool extensions in manifests. Plugins run
out of process or under the compile-time capability sandbox, declare inputs and
outputs, and exchange versioned schemas. A plugin cannot change grammar,
tokenization, visibility, or inject source text.
